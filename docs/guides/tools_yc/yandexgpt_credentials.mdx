# Получение доступа к YandexGPT

Для использования YandexGPT в ваших проектах необходимы два основных параметра:
- `folder-id` - идентификатор облачного каталога
- `api-key` (или `iam-token`) - ключ доступа к YandexGPT

## Предварительные требования

1. [Создайте Яндекс ID](https://passport.yandex.ru/auth/reg) (если у вас его еще нет)
2. [Подключите платежный аккаунт](https://yandex.cloud/ru/docs/billing/operations/create-new-account) в Yandex Cloud
3. При необходимости пополните баланс

## Получение ключа авторизации

### Способ 1: Использование API-ключа (рекомендуется)

API-ключ - это более простой способ авторизации, который также может быть безопасным при использовании временных ключей.

1. [Создайте сервисный аккаунт](https://yandex.cloud/ru/docs/iam/operations/sa/create)

2. Назначьте сервисному аккаунту необходимые роли:
   - `ai.languageModels.user` - для доступа к YandexGPT
   - `ai.speechkit-stt.user` - для распознавания речи (опционально)
   - `ai.speechkit-tts.user` - для синтеза речи (опционально)
   - `ai.imageGeneration.user` - для генерации изображений через YandexART (опционально)

3. [Создайте API-ключ](https://yandex.cloud/ru/docs/iam/concepts/authorization/api-key) для сервисного аккаунта

:::tip
Рекомендуется создавать API-ключи с ограниченным сроком действия для повышения безопасности.
:::

### Способ 2: Использование IAM-токена

IAM-токены предоставляют дополнительный уровень безопасности, так как имеют ограниченный срок действия (12 часов).

1. Создайте сервисный аккаунт (как описано выше)
2. [Получите IAM-токен](https://cloud.yandex.ru/docs/iam/concepts/authorization/iam-token)

## Получение folder-id

Идентификатор каталога можно получить одним из способов:
1. В графической консоли Yandex Cloud
2. В URL браузера при работе с консолью
3. Через CLI команду

Подробная инструкция доступна в [документации](https://yandex.cloud/en/docs/resource-manager/operations/folder/get-id).

## Использование учетных данных

### При использовании API-ключа

```python
headers = {
    "Authorization": f"Api-Key {api_key}",
    "x-folder-id": folder_id
}
```

### При использовании IAM-токена

```python
headers = {
    "Authorization": f"Bearer {iam_token}",
    "x-folder-id": folder_id
}
```

## Важные замечания

1. Храните учетные данные в безопасном месте
2. Не включайте их в код или системы контроля версий
3. Используйте переменные окружения или защищенные хранилища секретов
4. Регулярно обновляйте временные ключи и токены
